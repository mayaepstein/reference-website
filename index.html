Week 3: Naming Conventions & Commit Messages

Rules for naming files and paths

  • All lowercase
  • No spaces
  • Only letters, numbers, and dashes (-)

Commit Messages

Commit messages should be between 3-10 words. The should have proper grammar and spelling, a capital starting letter, and no periods. You should pretend that every commit starts with "This commit will..." and then add an imperative verb like create, fix, add, or solve.
