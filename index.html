Week 3: Naming Conventions & Commit Messages

Rules for naming files and paths

  • All lowercase
  • No spaces
  • Only letters, numbers, and dashes (-)

Commit Messages

Commit messages should be between 3-10 words. The should have proper grammar and spelling, a capital starting letter, and no periods. You should pretend that every commit starts with "This commit will..." and then add an imperative verb like create, fix, add, or solve.

Quotes, Citations, and Emphasis

Use a q tag for quotes that are short.
Use a blockquote tag for quotes that are longer.

Use a cite tag for Titles you want to cite

Use an em tag to emphasize text.

Use a strong tag for text that's important.

Use a b tag for text that you want to be bold.

Use an i tag to make text italic.

Use a small tag to make text smaller
